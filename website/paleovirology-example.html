A Hepadnaviridae-GLUE example for paleovirologists

This page provides a demonstration of Hepadnaviridae-GLUE being used to (i) organise the fossil record and; (ii) expand the fossil record to accomodate new data.

Once it had been established which category a set of one or more sequences belonged to, we processed it accordingly We used this resource to: (i) implement comparative analyses that are highly reproducible, and; (ii) make eHBV data available in a way that facilitates further investigation in this area.

We included reference information the hepadnavirus family, including reference genomes, alignments, trees, and other auxilliary data. For reference genomes, we used the library of reference genome sequences we constructed for DIGs (see above), linking these sequences with a record of their taxonomic and isolate-associated information (Table 1). A set of (coding and non-coding) genome features was defined, and the locations of these features on each reference sequence was recorded.

To incorporate EVEs the default data schema was extended to incorporate fields and tables representing the locations of genomic EVEs identified via DIGS, along with FASTA-formatted files representing the EVE sequences identified in the screen. We created a set of linked alignments that represented distinct taxonomic groups and levels within the family Hepadnaviridae. For the lower taxonomic levels (i.e. within and below genus level) we aligned complete coding sequences.

For higher taxonomic levels we aligned the polymerase or RT genes. We used GLUE to link these alignments, via a set of common reference sequences. The root alignment contains all reference sequences, whereas all children of the root inherit at least one reference from their immediate parent. Thus, all alignments are linked to one another via our chosen set of references.

The designation of a new locus was based on (i) phylogenetic grouping if this was clearly distinct from that of other eHBV elements, and (ii) the identification of unique flanking regions. Once a new eHBV locus had been identified, we used the DIGS framework to identify any additional orthologs of this insertion within the set of sequences previously obtained by screening.

Where appropriate, we also performed additional screens using the putative polypeptide gene products (“puteins”) of eHBV sequences. In this way, we systematically resolved the eHBV elements identified in our screen into orthologous sets of sequences.

Organising the fossil record

The presence of ancient orthologous eHBV insertions in certain vertebrate clades (e.g. Neoaves, Serpentes, Testudines) quickly became apparent once the genome sequences of multiple species were available for analysis - because these sequences were incorporated into the germline prior to the radiation of major host lineages, they are present at the same genomic locus in all species within those lineages. The Hepadaviridae-GLUE resource forms an integral part of our framework for recovering the hepadnavirus fossil record. Firstly, it provides a stable source of ‘probe’ and ‘reference’ sequences for database-integrated genome screening (DIGS) [19]. Other points: degradation of EVEs.
