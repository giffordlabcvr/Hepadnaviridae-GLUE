Endogenous hepadnavirus (eHBV) data

We have used GLUE to organise the fossil record of hepadnaviruses. You can access these data in this repository. This website provides quick links to specific data items.

eHBV reference sequences and data

We reconstructed reference sequences for ECVs by aligning eHBV sequences that derive from the same ancestral germline integration event.

Reference sequences were constructed by first constructing alignments of eHBV sequences derived from the same initial germline colonisation event - i.e. orthologous elements in distinct species, and paralogous elements that have arisen via intragenomic duplication of eHBV sequences.

Multiple sequence alignments and phylogenetic trees

Multiple sequence alignment constructed in this study are linked together using GLUE's ‘alignment tree’ data structure. Alignments in the project include:

(i) A single ‘root’ alignment constructed to represent proposed homologies between representative members of major CRESS virus lineages (including extinct lineages represented only by ECVs).
(ii) ‘genus-level’ alignments constructed to represent proposed homologies between the genomes of representative members of specific CRESS virus genera and eHBV reference sequences.
(iii) ‘Tip’ alignments in which all taxa are derived from a single virus species or represent distinct members of a single eHBV lineage.

We have used GLUE to set-up an automated process for deriving midpoint rooted, annotated trees for each of the alignments included in our alignment tree.

Raw eHBV sequences and data

These are the raw data generated by database-integrated genome screening (DIGS). The tabular files contain information about the genomic location of each EVE. EVEs were classified by comparison to a polypeptide sequence reference library designed to represent the known diversity of CRESS viruses - this includes extinct lineages represented only by endogenous viral elements (EVEs).

These data were obtained via DIGS performed in vertebrate genome assemblies downloaded from NCBI genomes (2020-07-15).

Nomenclature for ECVs

We have applied a systematic approach to naming eHBV, following a convention developed for endogenous retroviruses. Each element was assigned a unique identifier (ID) constructed from a defined set of components.

The first component is the classifier ‘eHBV’ (endogenous CRESS virus element).

The second component is a composite of two distinct subcomponents separated by a period: (i) the name of eHBV group; (ii) a numeric ID that uniquely identifies the insertion. The numeric ID is an integer identifies a unique insertion locus that arose as a consequence of an initial germline infection. Thus, orthologous copies in different species are given the same number. Where an eHBV lineage has been established via intragenomic amplication an incorporated eHBV sequence, distinct duplicates are identified by numbers after the decimal point.

The third component of the ID defines the set of host species in which the ortholog occurs.
